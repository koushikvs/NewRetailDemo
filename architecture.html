System Architecture · How the agent fabric is wired

One agent fabric. SAP at the core, Microsoft at the edge.

Every agent — internal to Joule, external in Azure AI Foundry, or anywhere else — registers via ORD, communicates via A2A, and operates inside a single Trust boundary. The pattern is identical whether the agent runs on SAP BTP, Microsoft Azure, Google Cloud, or AWS.

Why this pattern
Same registration (ORD), runtime (A2A), and trust contract everywhere. Switching agentic edges does not change SAP. Microsoft today, Google or AWS tomorrow — Joule and the Agent Catalog stay still.

Lean Core SAP S/4HANA + aATP
The system of record. Every wholesale PO, article master, season, allocation, condition record, and audit trail lives here. aATP drives all fulfilment logic — PAC, ABC, BOP, Supply Protection, Product Allocation, Season Management. The agent invokes aATP; it never bypasses it.
S/4HANA aATP · PAC / ABC / BOP MDG · RFM Season Mgmt Supply Protection VAS Conditions
Hybrid Edge SAP BTP / BDC
The integration fabric and agent runtime. Joule hosts orchestration agents that call S/4 + aATP. The Agent Catalog registers every agent — internal or external — via ORD, exposes a unified A2A invocation router, and federates identity across clouds. The Agent Connector serves ORD and A2A endpoints to external clouds. BDC is the unified semantic data layer.
BTP Subaccount Joule · Agent Builder Agent Catalog · CAP ORD Aggregator A2A Router SAP BDC HANA Cloud
Agentic Edge Microsoft
The agent runtime outside SAP. Azure AI Foundry hosts Copilot Studio + Azure OpenAI agents — drafting buyer communications, writing dispute letters, running three-way match. Microsoft Fabric ingests supplier mill telemetry, retailer POS, and weather/demand signals into ML-ready risk scores. Dynamics 365 is the retailer-360 + buyer context store. Power Automate is the workflow + portal-submission layer; Teams + Outlook is the human surface.
